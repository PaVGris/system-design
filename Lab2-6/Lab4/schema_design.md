# Проектирование документной модели MongoDB

## Коллекции

### 1. `items`

```json
{
  "_id": ObjectId("..."),
  "name": "Ноутбук Lenovo IdeaPad",
  "description": "15.6 дюймов, Intel Core i5, 8GB RAM",
  "price": 54999.99,
  "quantity": 42
}
```

### 2. `carts`

```json
{
  "_id": ObjectId("..."),
  "user_id": 1,
  "items": [
    { "item_id": "some_id_1", "quantity": 2 },
    { "item_id": "some_id_2", "quantity": 1 }
  ]
}
```

---

## Обоснование выбора embedded vs references

### `items` — отдельная коллекция (references из `carts`)

**Решение:** `carts.items[]` хранит только `item_id` (string) и `quantity`, а не полный документ товара.

**Обоснование:**
- Товар — независимая сущность с собственным жизненным циклом (создаётся, обновляется, удаляется отдельно от корзины)
- Один товар может быть в корзинах тысяч пользователей — дублировать данные нецелесообразно
- При изменении цены или названия товара не нужно обновлять все корзины
- Размер документа `cart` остаётся небольшим даже при большом количестве позиций

**Альтернатива (отклонена):** Embedded-документ с полными данными товара в корзине приведёт к дублированию данных и проблемам консистентности при обновлении цен.

### `carts.items` — embedded array

**Решение:** Список позиций корзины хранится как вложенный массив внутри документа корзины.

**Обоснование:**
- Позиция корзины не имеет смысла без корзины — это часть одной сущности
- Корзина всегда читается целиком (`GET /cart` возвращает все позиции сразу)
- Операции добавления (`$push`), обновления (`$inc`) и удаления (`$pull`) позиций эффективно работают с массивами в MongoDB
- Количество позиций в корзине ограничено (обычно десятки, не тысячи), что не создаёт проблем с размером документа (лимит 16MB)

**Альтернатива (отклонена):** Отдельная коллекция `cart_items` с полями `user_id`, `item_id`, `quantity` — реляционный подход, теряет преимущества документной модели и требует join-подобных операций.

## Индексы

```javascript
// items: поиск по названию
db.items.createIndex({ name: 1 })

// carts: поиск корзины по пользователю (основная операция)
db.carts.createIndex({ user_id: 1 }, { unique: true })

// carts: поиск конкретного товара в корзине
db.carts.createIndex({ "items.item_id": 1 })
```
